🔥 Step 1 — Firebase Database

Store all shop listings and customer orders in Firebase Firestore (free tier is enough to start)
What is Firebase?Google's free cloud database. No server needed — works directly from your website. Free up to 50,000 reads/day.
1

Create a Firebase project

Go to console.firebase.google.com → "Add project" → name it localmart → continue. Then go to Firestore Database → "Create database" → Start in test mode.

2

Get your config keys

In Firebase → Project Settings → scroll to "Your apps" → click Web (</>) → register app → copy the config object shown.

3

Add Firebase to your website

Paste this in your HTML file, replacing the config values with yours:

HTML — paste in <head>
<!-- Firebase SDK -->
<script type="module">
  import { initializeApp } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js";
  import { getFirestore, collection, addDoc,
           getDocs, query, where, orderBy }
    from "https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js";

  // ⬇ Replace with YOUR Firebase config
  const firebaseConfig = {
    apiKey:            "AIzaSy_YOUR_KEY_HERE",
    authDomain:        "localmart-xxxxx.firebaseapp.com",
    projectId:         "localmart-xxxxx",
    storageBucket:     "localmart-xxxxx.appspot.com",
    messagingSenderId: "123456789",
    appId:             "1:123:web:abc123"
  };

  const app = initializeApp(firebaseConfig);
  const db  = getFirestore(app);
  window.db = db; // make it global
</script>
4

Save a new shop to the database

Replace your registerShop() function with this:

JavaScript
import { collection, addDoc, serverTimestamp }
  from "https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js";

async function registerShop() {
  const name  = document.getElementById('r-name').value.trim();
  const owner = document.getElementById('r-owner').value.trim();
  const phone = document.getElementById('r-phone').value.trim();
  const cat   = document.getElementById('r-cat').value;
  const area  = document.getElementById('r-area').value;
  const addr  = document.getElementById('r-addr').value.trim();

  if (!name || !owner || !cat || !area) {
    alert('Please fill all required fields'); return;
  }

  try {
    await addDoc(collection(window.db, 'shops'), {
      name, owner, phone, cat, area, addr,
      rating:    0,
      approved:  false,   // admin approves before it goes live
      createdAt: serverTimestamp()
    });
    alert('✅ Shop registered! We will review and approve it shortly.');
  } catch (err) {
    alert('Error: ' + err.message);
  }
}

Load shops from the database

import { collection, getDocs, query, where }
  from "https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js";

async function loadShops() {
  // Only load approved shops
  const q = query(
    collection(window.db, 'shops'),
    where('approved', '==', true)
  );
  const snap = await getDocs(q);
  const shops = snap.docs.map(d => ({ id: d.id, ...d.data() }));
  renderShops(shops); // your existing render function
}

// Call on page load
window.addEventListener('load', loadShops);

Save orders to the database

JavaScript
async function placeOrder(shopId, shopName, cartItems, totalAmount, customerPhone) {
  const orderRef = await addDoc(collection(window.db, 'orders'), {
    shopId,
    shopName,
    items:         cartItems,
    total:         totalAmount,
    customerPhone,
    status:        'pending',
    paymentStatus: 'pending',
    createdAt:     serverTimestamp()
  });
  return orderRef.id; // use this as your order ID
}
Firestore data structure shops/ → {name, owner, phone, cat, area, approved, rating, createdAt}
orders/ → {shopId, shopName, items[], total, customerPhone, status, paymentStatus, createdAt}
users/ → {email, role: "shopkeeper"|"customer", shopId}

🔐 Step 2 — Login System

Firebase Authentication — customers and shopkeepers get their own accounts. Free, no server needed.
🛒
Customer login

Sign up with phone number or email. Can browse shops, place orders, track deliveries.

🏪
Shopkeeper login

Email login. Can manage their shop profile, add products, accept/reject orders.

1

Enable Authentication in Firebase

Firebase Console → Authentication → "Get started" → Enable Email/Password and Phone providers.

2

Add auth SDK and sign-up / login functions

JavaScript
import { getAuth, createUserWithEmailAndPassword,
         signInWithEmailAndPassword, signOut, onAuthStateChanged }
  from "https://www.gstatic.com/firebasejs/10.12.0/firebase-auth.js";

const auth = getAuth(app);

// ── SIGN UP (new shopkeeper) ──────────────────────────────
async function signUpShopkeeper(email, password, shopData) {
  const cred = await createUserWithEmailAndPassword(auth, email, password);

  // Save user role to Firestore
  await setDoc(doc(window.db, 'users', cred.user.uid), {
    email,
    role: 'shopkeeper',
    shopName: shopData.name,
    phone:    shopData.phone,
    createdAt: serverTimestamp()
  });
  return cred.user;
}

// ── LOGIN ─────────────────────────────────────────────────
async function login(email, password) {
  const cred = await signInWithEmailAndPassword(auth, email, password);
  return cred.user;
}

// ── WATCH login state (runs on every page load) ───────────
onAuthStateChanged(auth, async (user) => {
  if (user) {
    // User is logged in — fetch their role
    const snap = await getDoc(doc(window.db, 'users', user.uid));
    const role  = snap.data()?.role;
    if (role === 'shopkeeper') showShopkeeperDashboard();
    else showCustomerView();
  } else {
    showLoginScreen();
  }
});

// ── LOGOUT ───────────────────────────────────────────────
async function logout() {
  await signOut(auth);
}

Open payment popup when customer clicks "Pay Now"

JavaScript
function openRazorpay(orderData) {
  const options = {
    key:         "rzp_test_YOUR_KEY_ID", // from Razorpay dashboard
    amount:      orderData.total * 100,  // Razorpay uses paise (₹1 = 100)
    currency:    "INR",
    name:        "LocalMart",
    description: "Order from " + orderData.shopName,
    image:       "https://yourdomain.com/logo.png",

    // Prefill customer info
    prefill: {
      name:    orderData.customerName,
      contact: orderData.customerPhone
    },

    theme: { color: "#d4622a" },

    // ✅ Called when payment succeeds
    handler: async function(response) {
      const { razorpay_payment_id } = response;

      // Update order in Firebase as paid
      await updateDoc(doc(window.db, 'orders', orderData.orderId), {
        paymentStatus:  'paid',
        paymentId:      razorpay_payment_id,
        status:         'confirmed',
        paidAt:         serverTimestamp()
      });

      // Send WhatsApp alert to shopkeeper
      sendWhatsAppAlert(orderData);

      alert('✅ Payment successful! Order #' + orderData.orderId);
    },

    modal: {
      ondismiss: () => alert('Payment cancelled.')
    }
  };

  const rzp = new Razorpay(options);
  rzp.open();
}

📲 Step 4 — WhatsApp Order Alerts

Instantly notify shopkeepers on WhatsApp when a new order arrives. Two options below.
⚡
Option A — Simple link (free)

Opens WhatsApp on customer's phone pre-filled with order message. No API needed. Works immediately.

Free Manual send
🤖
Option B — Auto message (paid)

Uses Twilio or WATI API. Sends message automatically after payment. No customer action needed.

~₹2–5 per message Fully automatic

Option A — WhatsApp deep link (works right now, free)

JavaScript
function sendWhatsAppAlert(orderData) {
  const shopPhone = orderData.shopPhone.replace(/\D/g, ''); // digits only

  const itemsList = orderData.items
    .map(i => `• ${i.name} ×${i.qty} = ₹${i.price * i.qty}`)
    .join('\n');

  const message = `
🛒 *NEW ORDER — LocalMart*
Order ID: #${orderData.orderId}
Customer: ${orderData.customerName}
Phone: ${orderData.customerPhone}

*Items:*
${itemsList}

*Total: ₹${orderData.total}*
Payment: ✅ Paid via Razorpay

Please confirm the order!
`;

  const url = `https://wa.me/91${shopPhone}?text=`
            + encodeURIComponent(message);

  window.open(url, '_blank'); // opens WhatsApp
}

Option B — Automatic via Twilio API (₹2–5 per message)

1

Setup

Create account at twilio.com → activate WhatsApp Sandbox → get Account SID and Auth Token.

2

Send via a small backend (Node.js on Vercel — free)

api/notify.js — Vercel serverless function
const twilio = require('twilio');

const client = twilio(
  process.env.TWILIO_ACCOUNT_SID,
  process.env.TWILIO_AUTH_TOKEN
);

module.exports = async (req, res) => {
  const { shopPhone, message } = req.body;

  await client.messages.create({
    from: 'whatsapp:+14155238886', // Twilio sandbox number
    to:   `whatsapp:+91${shopPhone}`,
    body: message
  });

  res.json({ success: true });
};
